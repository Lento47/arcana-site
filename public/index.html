// Models

One terminal for every model key.

Arcana does not sell a model. It gives your existing provider keys a terminal-native session layer with memory, tools, and scheduled runs.

openai anthropic openrouter deepseek cerebras ollama local custom endpoint
// System

Everything around the model.

OpenRouter unifies model access. Arcana sits one layer above — the terminal where keys, tools, memory, jobs, and gateways work together.

01

Any model key

Add provider keys once and switch models by task, not by app.

02

Wiki-style memory

FTS5-powered session memory with extracted facts, full-text search, and optional cloud sync across machines.

03

Scoped tools

Run shell, filesystem, git, gateway, and skill tools through explicit profiles with visible permissions.

04

Skills on demand

Load specialized instructions only when needed to keep the base session clean.

05

Cron jobs

Schedule recurring prompts, reports, reviews, and checks from the same runtime.

06

Chat gateways

Route sessions to Telegram, Discord, Slack, or WhatsApp when the terminal is not the right surface.

07

Themes & backgrounds

7 arcane themes with custom background images for supported truecolor terminals.

08

Cloud memory sync

Push local facts to the cloud, pull on other machines, and share knowledge across your workspace.

// Start

Install locally. Add your key. Start a session.

No account is required for local use. Arcana should feel like a developer tool from the first run.

1

Install Arcana

Run Arcana locally from npm.

npx arcana-ai@latest
2

Activate your license

Add your Pro license when you need advanced features.

arcana license activate <key>
3

Start a session

Chat, run tools, save memory, or schedule a job.

arcana
