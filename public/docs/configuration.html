Configuration

Arcana is configured through a combination of a JSON config file and environment variables. The config file is loaded from arcana.json in your project root, or ~/.config/arcana/config.json as a fallback. Environment variables always take precedence over the config file.

Config File Reference

Create an arcana.json file in your project root. All fields are optional.

{
  "defaultModel": "openai/gpt-4o",
  "providers": {
    "openai": { "key": "sk-proj-..." },
    "anthropic": { "key": "sk-ant-..." },
    "deepseek": { "key": "sk-..." }
  },
  "tools": {
    "shell": { "enabled": true },
    "filesystem": { "enabled": true, "root": "./src" },
    "gateway": { "enabled": false }
  },
  "memory": {
    "enabled": true,
    "cloudSync": false
  },
  "theme": "bloodmoon",
  "proxy": {
    "url": "https://proxy.arcana.ai"
  }
}

Configuration Fields

FieldTypeDefaultDescription
defaultModelstringopenai/gpt-4oModel string used when none is specified
providersobject{}Map of provider names to API keys
toolsobject{}Per-tool configuration (enabled, permissions, root)
memory.enabledbooleantrueEnable persistent memory across sessions
memory.cloudSyncbooleanfalseSync memory to the cloud (requires license)
themestringdefaultTerminal theme name (see Themes)
proxy.urlstring—Custom proxy endpoint for model routing

Environment Variables

Environment variables override config file values and are the recommended way to set API keys (keeps them out of version control).

Arcana Settings

VariableDescription
ARCANA_THEMETerminal theme name (overrides config file)
ARCANA_PROXY_URLCustom proxy endpoint URL
ARCANA_LOG_LEVELLog verbosity: debug, info, warn, error
ARCANA_CONFIG_DIRCustom directory for config files

Config File Locations

Arcana looks for configuration in the following order (first match wins):

  1. ./arcana.json — project-level config
  2. ~/.config/arcana/config.json — user-level global config
  3. $ARCANA_CONFIG_DIR/config.json — custom path via env var

Values from all found config files are merged, with closer paths taking priority over global ones.
