ARCANA Documentation

Welcome to the official documentation for ARCANA — a terminal-native AI workbench that sits one layer above your model providers. This guide covers installation, configuration, core concepts, and advanced features.

Quick Start

Get up and running in under a minute. Install Arcana via npm and launch an interactive session with your first model provider:

# Install and launch in one command
npx arcana-ai@latest

# Or install globally for persistent access
npm install -g arcana-ai
arcana

The first run launches an interactive setup wizard that walks you through adding a provider key. Once configured, you can start a session with arcana from any terminal.

What is Arcana?

Arcana is a terminal-native AI workbench that decouples your model provider keys from the tools you use to interact with them. Instead of switching between ChatGPT, Claude.ai, and OpenRouter tabs, you work from a single terminal interface that provides:

  • Multi-provider sessions — Route individual prompts to different models (GPT-4, Claude, DeepSeek, Llama) from the same session, using the same toolchain.
  • Persistent memory — Facts extracted during a session are indexed and searchable across sessions via FTS5 full-text search. Optional cloud sync keeps your knowledge graph available across machines.
  • Scoped tool execution — Run shell commands, read and write files, interact with git, and invoke gateway APIs — all through explicit permission profiles that you control.
  • Scheduled automation — Define recurring prompts, daily summaries, code reviews, or health checks that run on a cron schedule from the same runtime.

Arcana does not sell model access. You bring your own API keys from any supported provider. The terminal layer adds session management, memory, tooling, and automation on top of your existing infrastructure.

Installation

Prerequisites

  • Node.js v18 or later (node --version)
  • An API key from any supported provider: OpenAI, Anthropic, DeepSeek, OpenRouter, Cerebras, or a local Ollama instance
  • A terminal with Unicode and truecolor support (iTerm2, Windows Terminal, Kitty, WezTerm, or any modern terminal emulator)

Install via npm

The recommended installation method uses npx, which requires no permanent installation:

npx arcana-ai@latest

For persistent access, install globally:

npm install -g arcana-ai
arcana --help

First Run

On first launch, Arcana prompts you to:

  1. Accept the license agreement
  2. Add at least one provider API key
  3. Select a model for your first session

After setup, you enter the interactive TUI (terminal user interface). Type /help to see available commands, or start typing naturally to chat with the model.

Core Concepts

Sessions

A session is an interactive conversation with one or more models. Sessions persist until you explicitly end them (/exit or Ctrl+D). Each session is recorded locally with full transcript, token usage, and cost tracking. Sessions made through the Arcana Proxy are also visible in the workspace dashboard.

Providers

Arcana supports any provider that exposes a chat-completion API. Add keys through arcana key add <provider> <key>. Supported providers include OpenAI, Anthropic, DeepSeek, OpenRouter (which aggregates 350+ models), Cerebras, Ollama (local models), and any OpenAI-compatible custom endpoint.

Memory

Arcana's memory system extracts structured facts from your conversations and indexes them using SQLite FTS5 full-text search. You can query your knowledge graph with arcana memory search <query>. The Memory documentation covers syncing, pruning, and cloud persistence.

Tools

Tools extend what the model can do: execute shell commands, read and write files, run git operations, and send messages through gateway integrations. Each tool category is governed by a permission profile — you approve or deny tool calls interactively. See MCP & Tools for details.

Skills

Skills are reusable instruction sets that you can load into any session. Instead of repeating complex instructions, you define a skill once (arcana skill add) and load it with /load <skill>. The Skills documentation covers authoring and sharing skills.

Configuration

Arcana stores configuration in ~/.arcana/config.yaml. You can view and modify settings through the CLI:

# View all configuration
arcana config --list

# Set a value
arcana config set editor.theme dracula

# Get a specific value
arcana config get editor.theme

The full configuration reference is available in the Configuration documentation.
