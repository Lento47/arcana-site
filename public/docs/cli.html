CLI Commands

Complete reference for all Arcana CLI commands. Run arcana --help in your terminal for a quick summary of available commands and flags.

Core Commands

arcana

Start an interactive AI session in your terminal. This is the default command when no subcommand is provided. Arcana will load your configured providers, memory, and tools, then present a prompt where you can ask questions, write code, or execute commands.

# Start a session with the default model
arcana

# Start with a specific provider and model
arcana --model anthropic/claude-sonnet-4-20250514

# Start in headless mode (no TUI)
arcana --headless

# Pass an initial prompt directly
arcana "Explain how React context works"
FlagDescription
--modelProvider/model string (e.g. openai/gpt-4o)
--headlessRun without the terminal UI
--no-toolsDisable all tool execution
--session <id>Resume an existing session by ID

arcana serve

Start the Arcana server, exposing an HTTP API that other tools and services can use to interact with Arcana. Useful for integrating with editors, CI pipelines, or custom workflows.

# Start on the default port (3000)
arcana serve

# Specify a custom port and host
arcana serve --port 8080 --host 0.0.0.0

# Enable verbose logging
arcana serve --verbose
FlagDefaultDescription
--port3000HTTP port to bind
--host127.0.0.1Interface to bind
--verbosefalseEnable verbose request logging

Configuration Commands

arcana config

View or modify Arcana configuration values. Configuration is persisted in arcana.json in your project root or ~/.config/arcana/config.json.

# View all configuration values
arcana config --list

# Get a specific value
arcana config get defaultModel

# Set a value
arcana config set theme bloodmoon

# Delete a config key (reset to default)
arcana config delete proxy.url

Key Management

arcana key

Manage AI provider API keys. Keys are stored encrypted at rest and never exposed in plain text through the CLI.

# Add a provider key
arcana key add openai sk-proj-xxxxxxxxxxxx

# List configured providers
arcana key list

# Remove a provider key
arcana key remove anthropic

# Test a key (verify it works)
arcana key test openai

Supported providers: openai, anthropic, deepseek, openrouter, google, cohere, mistral, groq, and any OpenAI-compatible provider.

Memory Commands

arcana memory

Manage Arcana's wiki-style memory system. Facts extracted during sessions are stored in a local SQLite database with full-text search (FTS5) and can optionally be synced to the cloud.

# Search memory for relevant facts
arcana memory search "deployment architecture"

# List all stored facts
arcana memory facts

# Filter facts by topic
arcana memory facts --topic "api"

# Clear all local memory
arcana memory clear

# Rebuild the FTS search index
arcana memory rebuild-index

Tools

arcana tools

List and manage the tools available to your session. Tools allow the AI to perform actions like running shell commands, reading files, or sending HTTP requests.

# List all registered tools
arcana tools list

# Check tool permissions and enabled state
arcana tools permissions

# Enable or disable a tool
arcana tools enable filesystem
arcana tools disable shell

Plugin Commands

arcana plugin

Install, list, and remove Arcana plugins. Plugins extend Arcana's capabilities — from custom tools to integrations with external services.

# Install a plugin from the registry
arcana plugin install arcana-mcp-fs

# List installed plugins
arcana plugin list

# Remove a plugin
arcana plugin remove arcana-mcp-fs

# Search the plugin registry
arcana plugin search "database"

Credits

arcana credits

Manage proxy credits for pay-as-you-go model routing through the Arcana proxy. Credits let you use the proxy without configuring your own provider keys.

# Check your credit balance
arcana credits balance

# Purchase additional credits
arcana credits buy 10

# View credit usage history
arcana credits history

License

arcana license

Manage your Arcana license key. Activating a license unlocks Pro or Enterprise features such as higher rate limits, cloud memory sync, and priority support.

# Activate a license key
arcana license activate arc_xxxx-xxxx-xxxx

# Check license status and expiry
arcana license status

# Deactivate the current license
arcana license deactivate

Global Flags

These flags work with any command:

FlagDescription
--helpShow help text for any command
--versionPrint the installed Arcana version
--verboseEnable verbose debug output
--jsonOutput results as JSON (useful for scripting)
