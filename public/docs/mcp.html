MCP & Tools

Arcana supports the Model Context Protocol (MCP), an open standard for connecting AI models to external tools and data sources. MCP servers let you extend Arcana with custom capabilities — database queries, API integrations, file operations, and more — all controlled through a unified interface.

How MCP Works

An MCP server exposes tools (functions the model can call), resources (data the model can read), and prompts (templates the model can use). Arcana connects to MCP servers at startup and makes their capabilities available to the AI during your session.

┌─────────────┐    MCP Protocol     ┌──────────────┐
│   Arcana    │ ──────────────────► │  MCP Server  │
│  (client)   │ ◄────────────────── │  (tools,     │
│             │                     │   resources) │
└─────────────┘                     └──────────────┘

Adding MCP Servers

You can connect Arcana to any MCP-compatible server using arcana mcp add.

# Connect to a local MCP server via HTTP
arcana mcp add my-tools --url http://localhost:3001

# Connect via Unix socket
arcana mcp add db-tools --url unix:///tmp/mcp.sock

# Connect to a remote server
arcana mcp add analytics --url https://mcp.example.com

# List connected servers
arcana mcp list

# Disconnect a server
arcana mcp remove my-tools

Tool Profiles

Control which tools are available in each session using tool profiles. Profiles let you restrict sensitive tools (like shell access) while allowing safe ones (like file reading). Configure profiles in arcana.json:

{
  "tools": {
    "shell": {
      "enabled": true,
      "allowCommands": ["git", "npm", "node"],
      "denyCommands": ["rm", "sudo", "curl"]
    },
    "filesystem": {
      "enabled": true,
      "root": "./src",
      "allowWrite": false
    },
    "gateway": {
      "enabled": false
    },
    "http": {
      "enabled": true,
      "allowedDomains": ["api.github.com", "api.openai.com"]
    }
  }
}

Tool Configuration Options

OptionTypeDescription
enabledbooleanWhether the tool is available
allowCommandsstring[]Allowed shell commands (shell tool only)
denyCommandsstring[]Blocked shell commands (takes priority)
rootstringRestrict filesystem access to this directory
allowWritebooleanAllow file write operations
allowedDomainsstring[]Restrict HTTP tool to these domains

Built-in Tools

Arcana ships with several built-in tools that are available by default:

  • shell — Execute shell commands (restricted by profile)
  • filesystem — Read and write files within a configured root
  • http — Make HTTP requests to allowed domains
  • memory — Search and store facts in persistent memory
  • gateway — Send messages through configured chat gateways
